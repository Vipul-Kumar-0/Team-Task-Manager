Build a complete full-stack web app called “Team Task Manager” for my assignment.

Goal:
Create a live, fully functional web application where users can create projects, manage teams, assign tasks, and track progress using role-based access control with two roles: Admin and Member.

Tech stack:
Use a simple production-ready full-stack setup:
- Frontend: React + Vite
- Backend: Node.js + Express
- Database: PostgreSQL
- ORM: Prisma
- Authentication: JWT + bcrypt password hashing
- Styling: clean responsive CSS or Tailwind
- Deployment-ready for Railway

Core features:

1. Authentication
- Signup
- Login
- Logout
- Passwords must be hashed with bcrypt
- JWT-based authentication
- Protected routes
- Show logged-in user name and role in the UI

2. Role-based access control
Admin users can:
- Create projects
- Edit/delete projects
- Add team members to projects
- Create tasks
- Assign tasks to team members
- Update any task
- View all projects and team progress

Member users can:
- View only projects they are assigned to
- View their assigned tasks
- Update task status only for tasks assigned to them
- Cannot create/delete projects
- Cannot manage users or assign tasks

3. Project management
Each project should include:
- Project title
- Description
- Created by admin
- Team members
- Created date
- Project status/progress based on task completion

4. Team management
- Admin can add existing users to a project
- Show project members on the project detail page
- Prevent duplicate members in the same project

5. Task management
Each task should include:
- Title
- Description
- Assigned user
- Project
- Status: Todo, In Progress, Completed
- Priority: Low, Medium, High
- Due date
- Created date
- Overdue indicator if due date has passed and task is not completed

6. Dashboard
Create a dashboard showing:
- Total projects
- Total tasks
- Completed tasks
- In-progress tasks
- Overdue tasks
- Task status summary
- Recent tasks
- For Admin: show all project/task stats
- For Member: show only their assigned work

7. Validations and relationships
Implement proper backend validation:
- Required fields
- Valid email format
- Unique email for users
- Valid task status and priority
- Due date validation
- Prevent unauthorized access
- Proper database relationships between User, Project, ProjectMember, and Task
